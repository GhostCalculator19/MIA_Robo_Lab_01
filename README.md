# II - Построение пайплайна (DVC) и обучение моделей

на данном этапе был создан файл [paramts.yaml](/Config/paramts.yaml), для настройки гиперпараметров моделей и также был создан файл [dvc.yaml](/dvc.yaml), имеющий структуру пайплайна для всех моделей.

Общая структура пайплайна DVC

```yaml

stages:

  # ================================ Prepare stage ================================
  preprocess:
    cmd: pагрузка подготовленных данных
    deps: Загрузка не подготовленных данных (raw)
    params:загрузка параметров
    outs: cоздаются файлы для обучения моделей (processed)
  
  # ================================ Train stages ================================
  train_model:
    cmd: запуск программы для обучения
    deps: Запуск обучающей модели
    params: загрузка параметров для подбора лучшей модели
    outs: Сохранение гистограмм весов и кривых обучения

  # ================================ Test stages ================================
  test_model:
    cmd: запуск программы для тестирования
    deps: Запуск тестирующей модели
    params: загрузка параметров
    metrics: Сохранение выходных метрик
    plots: Сохранение гистограмм весов и кривых обучения

```
В работе были реализованы 5 моделей и для каждого из них была написана такая же структура, как предствалено в [dvc.yaml](/dvc.yaml)

# **1. Реализация линейной регрессии**

**Линейная регрессия** - это метод анализа данных и алгоритм машинного обучения, который моделирует зависимость между одной или несколькими независимыми переменными (факторами) и зависимой переменной (целевым значением) в виде прямой линии.

# **2. Реализация дерева решений**

**Дерево решений** - модель, которая рекурсивно разбивает обучающую выборку по признакам, выбирая на каждом шаге разбиение с максимальным приростом информации. Каждый внутренний узел дерева проверяет один признак, ветви - варианты ответа, листья - итоговое предсказание.

Основные гиперпараметры дерева решений
- max_depth — максимальная глубина дерева
- min_samples_split — минимум объектов для разделения
- min_samples_leaf — минимум объектов в листе
- max_features — сколько признаков рассматривать

# **3. Реализация CatBoost**
**CatBoost** - это алгоритм градиентного бустинга,который строит ансамбль деревьев решений последовательно: каждое новое дерево исправляет ошибки предыдущих. Главная особенность является  Ordered Boosting - способ, позволяющий кодирования категиральные признаков и это решает проблему утечки данных.

Основные гиперпараметры CatBoost
- iterations — число деревьев
- learning_rate — скорость обучения
- depth — глубина деревьев
- border_count — число разбиений числовых признаков

# **4. Реализация XGBoost**
**XGBoost** - оптимизированная реализация градиентного бустинга. в отличии от **CatBoost**, при построении каждого дерева XGBoost использует второй порядок производной (гессиан) функции потерь, что позволяет точнее находить направление оптимизации.
Основные гиперпараметры XGBoost
- n_estimators — число деревьев
- learning_rate (eta) — шаг обучения
- max_depth — глубина деревьев
- subsample — доля данных
- colsample_bytree — доля признаков
- gamma — минимальное улучшение для сплита
- lambda, alpha — L2/L1 регуляризация

# **5. Реализация нейронной сети (Artificial Neural Network)**

Нейронная сеть (MLP) - это многослойная архитектура из нейронов, соединённых весами. Данные проходят вперёд (forward pass), вычисляется ошибка, и затем методом обратного распространения (backprop) + градиентного спуска корректируются веса. Способна улавливать сложные нелинейные зависимости.

Основные гиперпараметры MLP (Multi-Layer Perceptron)
- число слоёв
- число нейронов
- функции активауия - ReLU, sigmoid, tanh
- learning_rate - шаг/скорость обучения
- batch_size - размер батча/пакета
- epochs - количество эпох обучения
- оптимизаторы - Adam, SGD и др.
- dropout - регуляризация

# **6. Сравнительная таблица метрик качества моделей**

1) **RMSE (Root Mean Squared Error/ среднеквадратичная ошибка)** - это метрика, показывающая, насколько сильно в среднем предсказанные моделью значения отличаются от реальных (фактических) данных => **Минимальная** 

2) **MAE (Mean Absolute Error/ средняя абсолютная ошибка)** - это метрика, которая показывает средний модуль отклонения предсказанных значений от фактических в задачах регрессии, не смотря в какую сторону (в плюс или минус) была сделана ошибка => **Минимальная** 

3) **R2 (Коэффициент детерминации)** показывает, насколько хорошо построенная регрессионная модель описывает реальные данные, измеряя долю дисперсии зависимой переменной, объясненную независимыми переменными => **Максимальная**
